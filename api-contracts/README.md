# API Contracts - مرجع کامل ارتباط Backend و Flutter

این فولدر **منبع حقیقت واحد (Single Source of Truth)** برای تمام API‌های پروژه است.

## قوانین استفاده:
1. ✅ **هر تغییری در Backend ابتدا در این فایل‌ها ثبت شود**
2. ✅ **Flutter فقط طبق این مستندات پیاده‌سازی شود**
3. ✅ **هیچ endpoint بدون ثبت در اینجا پیاده‌سازی نشود**
4. ✅ **تمام DTOها و Response‌ها دقیقاً مطابق این فایل‌ها باشد**

## ساختار فایل‌ها:
- `auth-api.md` - تمام APIهای احراز هویت ✅
- `business-api.md` - تمام APIهای کسب‌وکار ✅
- `product-api.md` - تمام APIهای محصولات ✅
- `customer-api.md` - تمام APIهای مشتریان ✅ **NEW**
- `invoice-api.md` - تمام APIهای فاکتورها
- `common-types.md` - تایپ‌ها و مدل‌های مشترک ✅

## چک‌لیست قبل از پیاده‌سازی:
- [ ] آیا endpoint در فایل مربوطه ثبت شده؟
- [ ] آیا DTOها مطابقت دارند؟
- [ ] آیا Response format یکسان است؟
- [ ] آیا Error handling مشخص شده؟
